PBIX Analyzer

Case de Sucesso

Como o GitHub Copilot CLI nos ajudou a construir um app que documenta automaticamente relatórios Power BI e notebooks Databricks — transformando horas de trabalho manual em minutos de geração automática.

Python 3.14 PyInstaller GitHub Models gpt-4o GitHub Copilot CLI Power BI Databricks python-docx
O Problema

Documentação técnica é essencial — e ninguém quer fazer

😰

Antes — Processo Manual

  • Horas por arquivo para documentar estrutura e lógica
  • Documentação ficava desatualizada rapidamente
  • Estagiário novo não conseguia entender o notebook sozinho
  • Nenhum padrão entre relatórios e equipes diferentes
  • Análises críticas de pricing sem contexto de negócio registrado
🔥

O Cenário Real

Um notebook Databricks com 14 células, 5 queries SQL, 22 fontes de dados e UDFs complexas para extração de marca/modelo/família — sem nenhuma documentação.

Um relatório Power BI Pricing Evolution com 292 visuais, 149 campos, 21 tabelas e análise de GAP competitivo da John Deere — invisível para quem não construiu.

A Ideia

E se um agente de IA lesse o arquivo e escrevesse a documentação?

1

O app abre o arquivo (.pbix ou .ipynb)

Extrai toda a estrutura: páginas, medidas DAX, visuais, relações (PBIX) ou células, funções, SQL, imports, fontes de dados (notebook)

2

Envia cada elemento para o LLM (gpt-4o)

Para cada página do PBIX ou cada célula do notebook, um prompt especializado pede ao modelo que explique o propósito, as decisões técnicas e os riscos — como um engenheiro sênior documentaria

3

Gera um Word (.docx) estruturado e rico

Capa, índice, seções temáticas, código com tema Dracula, tabelas de métricas, narrativa LLM por página/célula — tudo em um clique

4

Tudo construído com GitHub Copilot CLI

O app inteiro foi desenvolvido em conversas com o Copilot CLI — desde a arquitetura até os bugs de produção. Sem abrir IDE, sem ambiente Python configurado.

A Solução

Um executável simples — sem instalação

🖥️

Interface Tkinter

Aba PBIX e aba Notebook. Campo de seleção, log em tempo real, aba de config LLM. App empacotado como .exe via PyInstaller.

📊

Analisador PBIX

Abre .pbix como ZIP. Extrai medidas DAX, visuais, relações, tabelas, tooltips. LLM gera narrativa por página.

🐍

Analisador Notebook

Parseia células, detecta magics %sql %pip %sh, extrai funções, SQL, UDFs. LLM gera 3 caixas por célula.

🤖

GitHub Models / gpt-4o

Token pessoal GitHub. max_tokens=2000 por célula. Resposta em JSON estruturado: contexto · detalhe · atenção.

🎨

Tema Dracula

Código nos documentos com syntax highlight usando pygments. Keywords, strings, funções e comentários cada um com sua cor.

📦

Deploy SharePoint

PBIX_Analyzer.exe publicado em dois caminhos SharePoint sincronizados. Sem Python. Sem dependências. Abre e usa.

Guia de leitura gerado pelo LLM:

Células 1–3: Verificar bibliotecas e dependências instaladas no cluster (msal, pandas, pyspark).
Células 4–6: Analisar as UDFs _n, _get_marca, _strip_noise — fundamentais para normalização.

Células 10–14: Exploração e validação dos dados transformados.

📋 Contexto — Célula 4 (UDFs de Extração)
Esta célula define as funções-core do pipeline: _brand_from_text, _family_from_text e _model_for_brand. Elas substituem toda a lógica SQL de CASE/WHEN por matching brand-aware via catálogo catalogo_geral_48_fabricantes_v3.xlsx.
🔍 Detalhe Técnico
O dicionário _BRAND_ALIASES mapeia 48 fabricantes com variações ortográficas para formas canônicas — evitando fragmentação de dados por erros de digitação ("JHON DEERE", "J DEERE", "JDB" → "JOHN DEERE").
⚠️ Atenção
O catálogo catalogo_geral_48_fabricantes_v3.xlsx é carregado de um Volume Databricks. Se o arquivo for movido ou renomeado, todas as UDFs falham silenciosamente.
📋 Contexto — Célula 7 (Criação da Tabela Silver)
Cria/substitui a tabela jd.ds_silver_jd.silver_empresometro na camada silver a partir da tabela refinada, aplicando todas as transformações anteriores.
Operação DROP + CREATE — qualquer execução sobrescreve os dados anteriores sem confirmação.

Arquitetura

Como o app é construído

pbix_analyzer_gui.py

Interface Tkinter. Gerencia abas, log em tempo real, config LLM. Salva config em config.json no APP_DIR (SharePoint).

pbix_analyzer.py

Core PBIX. Abre como ZIP, extrai DAX/visuais/relações. Função call_llm() central. Gera .docx formatado.

ipynb_analyzer.py

Core notebooks. Parseia células, magics SQL/pip/sh, extrai UDFs. Usa _call_llm_notebook() com max_tokens=2000.

pygments (Dracula)

Tokeniza código e aplica cores por tipo de token no Word. Keywords, strings, funções, comentários com paleta Dracula.

PyInstaller .spec

excludes=['ipynb_analyzer'] para evitar bytecode stale. ipynb_analyzer.py como DATA file em _MEIPASS.

GitHub Models API

Token GitHub Personal Access Token. Endpoint Azure OpenAI-compatible. JSON por célula: contexto · detalhe · atenção. Logging completo de erros.

Como replicar

Prompts para criar um app similar com Copilot CLI

Crie um app Python desktop (.exe via PyInstaller) com Tkinter que analisa arquivos [TIPO: .pbix/.ipynb/.xlsx] e gera documentação em Word (.docx). O app deve ter: - Aba de seleção de arquivo e pasta de saída - Log em tempo real na interface - Integração com GitHub Models (gpt-4o) via Personal Access Token - Para cada [elemento extraído], chamar LLM retornando JSON: {"contexto": "...", "detalhe": "...", "atencao": "..."} Estruture em 3 arquivos: gui.py, analyzer.py e SPEC para PyInstaller.
